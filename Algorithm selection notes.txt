CHOICE 1: A* SEARCH W/ ITERATIVE DEEPENING (8 points)
--------------------------------------------------------------
A* search uses an evaluation function to estimate the distance to the goal node.
Certain nodes are prioritised over others.
A* search (& w/ iterative deepening) guarentee all nodes within a certain depth are explored).
The heuristic measure is the estimated cost to get from a node on the fringe to a goal state.

Iterative deepening performs a depth-limited search: It starts at a shallow depth and gradually increases depth limit w/ each iteration until goal is found.
At each iteration, the depth limit is set to a predefined amount to prevent the search going too deep in the search space.
This reduced memory requirements.
It's used with A* search by using the evaluation function to prioritise nodes within the depth limit to explore.
It may have increasing overhead but works well when depth of solution is unknown or varies.

These 2 things should be implemented SEPERATELY. This will allow debugging and modular design, plus give me a solid algorithm to fall back on if I run out of time.

Definitions:
- A* Score: For each node, the cost of path from source to that node + heuristic (estimated cost from that node to target)

Need to plan data structures. Information captured:
- Visited: A list of visited nodes + their A* score
- Best possible tour so far (may need to terminate within 1 min)
- Heuristic function: Should satisfy triangle equality - estimated cost from one city to another city plus the estimated cost from that 
  city to the goal should be less than or equal to the estimated cost from the initial city to the goal. 
  Minimum spanning tree heuristic possible.
